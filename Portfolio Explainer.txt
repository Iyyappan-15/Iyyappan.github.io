  3. WHY WERE THESE TECHNOLOGIES CHOSEN?
================================================================================

Q: Why plain HTML, CSS, and JavaScript instead of React, Next.js, etc.?

A: A personal portfolio is a static content site — it does not need a
   JavaScript framework. Using plain HTML/CSS/JS means:

   - FASTER load times (no framework bundle to download)
   - ZERO build step required — files work directly in the browser
   - EASIER to deploy on free services like GitHub Pages / Cloudflare Pages
   - EASIER for recruiters and developers to read the source code directly
   - Lower complexity = fewer bugs and easier maintenance

   Frameworks like React are ideal for dynamic apps with state management,
   routing, and component reuse at scale. For a portfolio with 6 pages and
   static content, they would add unnecessary overhead.

Q: Why Cloudflare Pages for hosting?

A: - It is 100% free for personal projects
   - Automatically deploys on every `git push` to GitHub (CI/CD built-in)
   - Provides a global CDN — the site loads fast from any country
   - Supports custom domains with free HTTPS/SSL certificates
   - Build times are under 60 seconds for a static site

Q: Why Space Grotesk and Fira Code fonts?

A: - Space Grotesk gives a modern, techy, and professional look — widely
     used in developer portfolios and SaaS products
   - Fira Code is a monospace font designed for code editors — using it for
     labels like "01 / 07" and section identifiers gives a developer feel
   - Both are loaded from Google Fonts CDN for reliability and speed

Q: Why CSS custom properties (variables) instead of hardcoded colors?

A: The site supports both Light Mode and Dark Mode. CSS variables allow
   all colors to be defined in one place (:root for light, .dark-theme
   for dark) and automatically switch across the entire site with one
   class toggle. This avoids duplicating hundreds of color rules.

Q: Why use SVG icons instead of an icon library (FontAwesome, etc.)?

A: - Inline SVGs have zero dependency — no extra HTTP request for an icon font
   - They scale perfectly at any size (vector, not pixel-based)
   - Color can be controlled via CSS `currentColor` property
   - They are accessible and semantic when given proper labels

  6. WHAT IS THE DESIGN PHILOSOPHY?
================================================================================

Q: What visual theme does the portfolio use?

A: The portfolio uses a Glassmorphism-inspired design with:
   - A dark mode by default with deep navy/black backgrounds
   - Light mode with clean white/gray backgrounds
   - Cyan (#00e5ff in dark / #0284c7 in light) and Purple (#8b5cf6)
     as accent colors — giving a modern, techy feel
   - Frosted glass card effects using rgba() transparency and backdrop blur
   - Smooth CSS transitions on all interactions

Q: How is dark/light mode implemented?

A: A JavaScript toggle button checks localStorage for the saved preference.
   On click, it adds/removes the `dark-theme` class on the <body> element.
   All colors are CSS variables that change based on which class is active.
   The preference is saved to localStorage so it persists across page visits.

Q: How are animations implemented?

A: - Scroll reveal animations: An IntersectionObserver watches elements with
     the `.reveal` class. When they enter the viewport, `.visible` is added,
     triggering a CSS `opacity + translateY` transition.
   - Particle background (Home page): A HTML5 Canvas animation draws floating
     cyan and purple dots that drift slowly across the hero section.
   - Hover effects: CSS `transform: translateY(-5px)` and `box-shadow` changes
     on project cards give a lift effect on hover.

  8. HOW IS THE SITE DEPLOYED? (WORKFLOW)
================================================================================

Step 1: Code is written and edited locally on Windows
Step 2: Changes are committed using Git
Step 3: `git push` sends changes to GitHub repository
         (github.com/Iyyappan-15/Iyyappan.github.io)
Step 4: Cloudflare Pages automatically detects the push
Step 5: Cloudflare builds and deploys the site globally (< 60 seconds)
Step 6: Site is live at https://iyyappan.me

This is a fully automated CI/CD pipeline — no manual upload required.


================================================================================
  9. IS THE SITE RESPONSIVE?
================================================================================

Yes. The site is fully responsive across all screen sizes:

  - Desktop  (1200px+) : Multi-column grid layouts
  - Tablet   (768px+)  : Two-column or adjusted layouts
  - Mobile   (<768px)  : Single-column, hamburger menu navigation

CSS media queries in style.css handle the breakpoints.
The hamburger menu on mobile is implemented using a CSS-only checkbox trick
(no JavaScript required for the menu toggle).
